# Optimizing Macro Asset Allocation for Mandatory Provident Funds (MPF)

这份项目的目标是：
用 Python 把一个“养老基金资产配置”问题，变成一个可以计算、可视化、可解释的完整流程。

如果你完全不懂代码，也可以把这份 README 当成“中文翻译版流程图”。
它会把 Notebook 里每一段代码在做什么、为什么要这么做、输出结果代表什么，全部讲清楚。

---

## 1. 这个项目在解决什么问题？

你可以把它理解成一个现实问题：

“假设我只能投这三类资产（美股、港股、债券），过去 10 年数据告诉我，怎么配比更合理？”

本项目用到的三类资产：

- `^GSPC`：标普 500 指数（代表美国股票市场）
- `^HSI`：恒生指数（代表香港股票市场）
- `AGG`：美国综合债券 ETF（代表债券市场）

核心输出有 3 类：

- 历史表现图（谁涨得快、谁波动大）
- 优化结果（最大夏普比组合、最小波动组合）
- 目标收益对应的风险评估（你想要多少收益，要承担多大风险）

---

## 2. 代码总流程（先有全局图，再看细节）

Notebook 一共 9 个代码部分，逻辑顺序如下：

1. 导入工具库
2. 设定资产和时间范围
3. 下载并清洗价格数据
4. 画标准化走势对比图
5. 计算收益率与风险矩阵
6. 蒙特卡洛随机组合模拟
7. 数学优化求最优组合
8. 画有效前沿和最优点
9. 输入目标收益，输出“个性化组合 + 压力测试”

一句话总结：
先看历史数据 -> 再做风险收益建模 -> 再做组合优化 -> 最后给出可落地的投资决策解释。

---

## 3. 每一部分代码到底在做什么（超详细）

下面按 Notebook 的顺序解释。

### 第 1 部分：导入库（工具箱）

导入了这些包：

- `yfinance`：从 Yahoo Finance 下载历史行情
- `pandas`：处理表格数据
- `matplotlib`：画图
- `numpy`：做数值计算
- `scipy.optimize`：做约束优化（找最优权重）
- `scipy.stats`：统计分布相关（本项目中用于风险解释扩展）

为什么需要？
因为这套项目不是只画图，而是完整走完“数据 -> 模型 -> 优化 -> 解释”的流程。

---

### 第 2 部分：定义研究范围

你告诉程序三件事：

- 投哪些资产（3 个 ticker）
- 从什么时候开始（2016-01-01）
- 到什么时候结束（2026-03-01）

这一步就像做实验先写“样本范围”。

---

### 第 3 部分：下载与清洗数据

这部分做了 4 件关键事情：

1. 从 Yahoo Finance 下载三类资产历史价格
2. 兼容不同 `yfinance` 版本：
   优先使用 `Adj Close`，如果没有就自动用 `Close`
3. 用前向填充 `ffill()` 处理缺失值（沿用前一个交易日价格）
4. 删除仍有缺失的日期，只保留三资产都齐全的数据

为什么很重要？

- 没有统一清洗，后面收益率和协方差都会失真
- 如果某个资产缺一天，组合计算就会对不齐

输出：

- `clean_data`：一张“日期 × 资产价格”的干净表格

---

### 第 4 部分：标准化走势可视化

原始价格不能直接比，因为指数和 ETF 的绝对价格单位不同。

所以代码做了标准化：

`normalized_data = (clean_data / clean_data.iloc[0]) * 100`

含义：

- 把每个资产起点都设为 100
- 后面数值表示“相对起点涨了/跌了多少”

图上功能点：

- `plt.legend()`：显示图例，让你知道每条线是谁
- `plt.tight_layout()`：自动调整边距，避免标题、坐标轴、图例重叠或被裁切

输出：

- 一张非常适合放报告的“过去 10 年三类资产相对表现图”

---

### 第 5 部分：把“价格”变成“收益与风险”

这里完成金融建模的底层输入：

1. 先算日收益率：
   `returns = clean_data.pct_change().dropna()`
2. 年化收益率：
   `annual_returns = returns.mean() * 252`
3. 年化协方差矩阵：
   `annual_cov_matrix = returns.cov() * 252`

为什么用 252？

- 一年大约 252 个交易日，是金融实务里常见换算。

协方差矩阵有什么用？

- 它描述“资产之间一起波动的关系”
- 后面组合风险计算完全依赖它

---

### 第 6 部分：蒙特卡洛模拟 10,000 个随机组合

这部分本质上是“先把可能性空间扫一遍”：

1. 随机生成很多组权重（每组权重和为 1）
2. 对每组算：
   - 组合预期收益
   - 组合波动率（风险）
   - 夏普比率（单位风险换来的超额收益）
3. 用散点图画出来

图怎么读？

- 横轴：风险（波动率）
- 纵轴：预期收益
- 颜色：夏普比率高低

你会看到：

- 点云代表“各种随机配置可能结果”
- 风险和收益通常存在权衡

---

### 第 7 部分：数学优化找两个经典最优组合

定义了三个函数：

- `calc_portfolio_perf`：给定权重，返回组合收益和波动率
- `neg_sharpe`：负夏普比率（因为优化器默认做“最小化”，所以取负号）
- `get_volatility`：返回组合波动率

约束条件：

- 所有权重之和必须等于 1
- 每个资产权重在 0 到 1 之间（不允许做空）

优化出了两种组合：

1. **最大夏普组合（更进取）**
2. **全局最小波动组合（更保守）**

输出表格把权重转成百分比，便于报告展示。

---

### 第 8 部分：有效前沿（Efficient Frontier）

这部分在做“理论最优边界线”：

1. 设定一串目标收益
2. 对每个目标收益，求在该收益下“风险最小”的组合
3. 把这些最小风险点连成线，就是有效前沿

同一张图里还标出：

- 随机组合散点云
- 有效前沿（红色虚线）
- 最大夏普点（红色星标）
- 最小波动点（绿色星标）

这张图非常关键，因为它把“可行组合”和“最优边界”同时呈现。

---

### 第 9 部分：目标导向投资 + 压力测试

这是最贴近实际决策的部分。

用户输入一个目标年化收益（例如 0.06 代表 6%），程序会：

1. 检查目标是否在历史可行范围内
2. 如果可行，求“达到该收益的最小风险组合”
3. 输出该组合权重
4. 输出风险解释（波动率）
5. 给出 95% 信心水平下的“最差年份估计”

最差情景用的是：

`worst_case = target_return - 1.645 * minimized_vol`

这里的 1.645 来自正态分布 95% 单侧分位数，
意思是：
“在较差但并非极端到 1% 尾部的年份里，回报可能下探到哪里”。

这段输出非常适合报告中的“风险提示”和“投资者沟通语言”。

## 4. 你朋友写报告时可以怎么写（可直接套用）

下面给一个实用的报告结构模板：

1. **研究问题**
   本研究在三类宏观资产（美股、港股、债券）之间进行配置优化，目标是在给定约束下寻找更优风险收益组合。

2. **数据与样本**
   数据来自 Yahoo Finance，样本期为 2016-01-01 至 2026-03-01，使用收盘相关价格并进行缺失值清洗。

3. **方法**
   使用日收益率推导年化收益与协方差矩阵；通过蒙特卡洛模拟展示可行组合分布；再用 SLSQP 在不做空约束下求解最大夏普组合和最小波动组合，并构建有效前沿。

4. **结果展示**
   - 标准化走势图：比较三资产历史累计表现
   - 风险收益散点图：展示随机组合分布
   - 有效前沿图：展示最优边界与两个代表性最优组合
   - 目标收益情景：展示给定收益目标下的最小风险权重和压力测试

5. **结论与含义**
   - 高收益目标通常要求更高风险暴露
   - 分散配置可在一定程度上改善风险收益比
   - 投资决策应结合目标收益、可承受回撤和风险偏好

6. **局限性**
   - 基于历史数据，不能保证未来重复
   - 假设收益分布近似稳定，现实中可能出现结构性变化
   - 未考虑交易成本、税务和流动性冲击
